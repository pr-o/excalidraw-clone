# Frame Labels & Membership Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rendered, double-click-editable frame names plus live membership: frame drag moves members, membership re-evaluates on every mutation, deleting a frame releases members.

**Architecture:** Membership becomes a scene invariant (`reconcileFrameMembership` in `Scene.mutate`), replacing the frame tool's one-shot `populateFrameMembers`. Drag expansion mirrors the group pattern (`expandIdsToFrameMembers`). Rename reuses the `startTextEdit` effect with a frame branch in `TextEditingOverlay` and a `renameFrame` driver. Both renderers draw `name ?? "Frame"` above the frame's top-left corner.

**Tech Stack:** TypeScript monorepo (pnpm + turbo), vitest, Playwright. Spec: `docs/superpowers/specs/2026-07-19-frames-design.md`.

## Global Constraints

- Membership rule: topmost (last in scene order) non-deleted frame fully containing the element's bounds (`boundsContains(frameBounds, getElementBounds(e))`), else `null`. Skip frames themselves and bound labels (`type === "text" && containerId !== null`). Reference-stable no-op writes.
- Selecting a frame never selects its members; only the drag translate set is expanded.
- Name rendering: `name ?? "Frame"`, 12px, `fontSpec(12, 2)`, fill `resolveColor("#868e96", theme)`, element-local `(0, -4)`, bottom baseline.
- Rename: trim; empty → `null`; unchanged → `skipHistory` mutation; Escape = no mutation.
- `exactOptionalPropertyTypes` is on — new optional interface props receiving possibly-undefined values need `| undefined`.
- RTK: `>/dev/null 2>&1 && echo PASS || echo FAIL`; `rtk proxy` for details. Lint from repo root. Commits: `<package>: <what>` + `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`.
